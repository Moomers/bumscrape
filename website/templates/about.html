TL;DR

Scalpers could wreck burning man, we should keep an eye on this.

So what is this?

BumScrape is a program that collects data about Burning Man tickets being resold on the web. BumScrape periodically visits popular ticket-vending websites; we currently check Craigslist, eBay and StubHub. Whenever we find a BM ticket listing on one of these sites, we log information about the listing in a database. We track each listing for as long as it's posted. The goal is to get an idea of the size and properties of the external resale market in BM tickets.

What happens to the data you gather?

All our data is public. Check out all of the active listings we're currently tracking here. You can get a copy of our archived data here.

Please don't use this site to buy a scalped ticket! Paying over face-value for a ticket encourages more scalping and commoditizes our community.

How does it work?

We used the Scrapy web-scraping framework for Python. We wrote spiders for each of the sites we're interested in. Those spiders generate items which we validate and then save into a MySQL database via an item pipeline. So far, only our Craigslist crawler is a traditional web scraper—we also use the APIs provided by StubHub and eBay, but we wrap those API calls in spider semantics.

How can I help?

We could use coding help to create more spiders. If you know of a ticket-vending site we're not aware of, let us know, or write a spider for it and send us a patch.

I want to gather the data myself! How do I use your software?

The code is available via git:

git clone http://igor.moomers.org/git/bumscrape

To crawl, you'll need to install Python and scrapy. If you want to save the data, you'll need a MySQL db and the mysqldb module. If you want to use a different database, you can replace the Store class in store.py. To use the eBay API, you'll need to get an eBay developer account. It's free, but you're limited to 5k queries per day unless you work with them to get your app certified.

This website uses web.py, a lightweight python web framework. We are running on Apache 2 and using modwsgi. If you want to run a similar site, you can use our apache config as a starting point.

Why are you doing this?

Many of us at Camp Warp Zone didn't win tickets in the 2012 ticket lottery. Like many burners, we were curious why so few tickets ended up in the hands of the people we knew in the community. We thought of three possible scenarios:

  1. Many burners bought too many tickets to increase their odds, and were embarrased to come forward.
  2. Lots of burgins not affiliated with any theme camps bought all the tickets.
  3. Scalpers beat the BMOrg and are sitting on a ticket bonanza.

Probably the truth is somewhere in between. We aren't sure about the first two scenarios, but we're especially worried about the third. BumScrape is an attempt to see how widespread scalping is. Scalpers cannot sell tickets in secret—they do it out in the open, on the web. Let's keep tabs on them!
