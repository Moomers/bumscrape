What is BumScrape

BumScrape is a web-crawling scraper which collects data about Burning Man tickets scalped on the Internet. BumScrape periodically visits popular ticket-vending websites; we currently check Craigslist, eBay and StubHub. Whenever we find a BM ticket listing on one of these sites, we log information about the listing in a database. We will keep track of how the listing progresses for as long as the listing is visible on the site where we found it. The goal is to get an idea of the size and properties of the external resale market in BM tickets.

What happens to the data you gather?

We are happy to let you see our data! Check out all of the active listings we're currently tracking here. You can get a copy of our archived data here.

Please don't use this site to buy a scalped ticket! Paying over face-value for a ticket encourages more scalping and commoditizes our community.

How does it work?

We used the Scrapy web-scraping framework for Python. We wrote spiders for each of the sites we're interested in. Those spiders generate items which we validate and then save into a MySQL database via an item pipeline. So far, only our Craigslist crawler is a traditional web scraper -- we also use the APIs provided by StubHub and eBay, but we wrap those API calls in spider semantics.

How can I help?

We could probably use more spiders. If you know of a ticket-vending site we're not aware of, write a spider for it and send us a patch.

I want to gather the data myself! How do I use your software?

Our code is available via git; simply do a git clone http://igor.moomers.org/git/bumscrape.

To crawl, you'll need to install python and scrapy. If you want to save the data, you'll need a MySQL db and the mysqldb module. If you want to use a different DB, you can replace the Store class in store.py. To use the eBay API, you'll need to get an eBay developer account. It's free, but you're limited to 5k queries per day unless you work with them to get your app certified.

This website uses web.py, a lightweight python web framework. We are running on Apache 2 and using modwsgi. If you want to run a similar site, you can use our apache config as a starting point.

Why are you doing this?

Many of us at Camp Warp Zone didn't win tickets in the 2012 ticket lottery. Like many burners, we were curious why so few tickets ended up in the hands of the people we knew in the community. We thought of three possible outcomes:

  • Many burners bought too many tickets to increase their odds, and were embarrased to come forward
  • Lots of burgins not affiliated with any theme camps bought all the tickets
  • Scalpers beat the BMOrg and were sitting on a ticket bonanza

We don't have enough data to speak about the first two possibilities. Possibly, the BMOrg does, but they're very unlikely to release the data given their MO. BumScrape is an attempt to explore the third possibility. Scalpers cannot sell tickets in secret -- they do it out in the open, on the internet. Lets keep tabs on them!
